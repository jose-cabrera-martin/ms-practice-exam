📋 Formato de preguntas — pipe |
Delimitador principal | — sin ambigüedad con las comas dentro de las opciones.

Una pregunta por línea:
ID:01|TYPE:"MULTI"|QUESTION:"texto"|OPTIONS:"A","B","C","D"|ANSWER:"A","C"|EXPLANATION:"texto"|POINTS:1

Campos:   ID único  ·  TYPE insensible a mayúsculas  ·  OPTIONS cuatro opciones entre comillas separadas por coma  ·  ANSWER letras correctas entre comillas  ·  EXPLANATION y POINTS opcionales (defecto 1).
TYPE = "SINGLE"
ANSWER = una sola letra: "A" "B" "C" "D"

ID:01|TYPE:"SINGLE"|QUESTION:"¿Qué es Copilot?"|OPTIONS:"IA generativa","Antivirus","BBDD","Firewall"|ANSWER:"A"|EXPLANATION:"Copilot es IA generativa de Microsoft."|POINTS:1
TYPE = "MULTI" (o "MULTIPLE")
ANSWER = varias letras entre comillas: "A","C" o "A","C","D"
Indica el nº a elegir en el enunciado: (Choose 2), (Elige 3).

ID:02|TYPE:"MULTI"|QUESTION:"Elige 2 buenas prácticas (Choose 2)"|OPTIONS:"Ser vago","Especificar intención","Dar ejemplos","Solo 1 palabra"|ANSWER:"B","C"|EXPLANATION:"Especificidad y ejemplos mejoran resultados."|POINTS:1
TYPE = "FILL" / "DROPDOWN" / "CLOZE" / "COMPLETE"
La pregunta debe contener {{BLANK}} donde aparece el selector.
Si no contiene {{BLANK}} → se convierte automáticamente a multi o single.

ID:03|TYPE:"FILL"|QUESTION:"Copilot opera dentro de los límites de {{BLANK}} de M365."|OPTIONS:"Azure AD","Compliance","SharePoint","OneDrive"|ANSWER:"B"|EXPLANATION:"Compliance garantiza seguridad."|POINTS:1
TYPE = "YESNO" (también: "SINO", "TRUEFALSE", "TF")
STATEMENTS = tres afirmaciones entre comillas separadas por coma.
ANSWER = "Sí" o "No" por cada afirmación, en el mismo orden.
Puntuación: todo-o-nada (las tres deben ser correctas).

ID:04|TYPE:"YESNO"|QUESTION:"Sobre Copilot en Microsoft 365:"|STATEMENTS:"Copilot accede a internet en tiempo real","Copilot usa datos del tenant","Copilot puede reemplazar al humano"|ANSWER:"No","Sí","No"|EXPLANATION:"Copilot usa datos del tenant pero no navega por internet."|POINTS:1
TYPE = "DRAG" (también: "ORDER", "SORT", "ORDENAR")
ITEMS = pasos en el orden correcto; el sistema los mezcla al presentarlos.
Puntuación: todo-o-nada. Arrastrar con ratón o toque (móvil).

ID:05|TYPE:"DRAG"|QUESTION:"Ordena las fases de despliegue de Copilot:"|ITEMS:"Identificar casos de uso piloto","Configurar permisos y governance","Habilitar licencias","Medir adopción y ROI"|EXPLANATION:"Casos de uso → governance → licencias → medición."|POINTS:1
TYPE = "MATCH"
STATEMENTS = las frases de la izquierda (entre comillas, separadas por coma).
OPTIONS = el pool de respuestas (entre comillas, separadas por coma).
ANSWER = las etiquetas (A, B, C...) correspondientes a cada statement.

ID:06|TYPE:"MATCH"|QUESTION:"Relaciona el servicio con su nube:"|STATEMENTS:"Azure SQL","EC2","S3","Office 365"|OPTIONS:"Microsoft Azure","Amazon Web Services"|ANSWER:"A","B","B","A"|EXPLANATION:"SQL y Office son de MS, EC2 y S3 de AWS."|POINTS:1
